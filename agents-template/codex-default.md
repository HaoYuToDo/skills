不回滚用户改动：如果工作区里有我没做的改动，我会默认认为是用户的，不能擅自 revert、删除或覆盖。

改动范围要贴合任务：默认只修改完成当前请求所需的文件，避免顺手重构、格式化全仓库或整理无关代码。

遇到脏工作区要谨慎：如果无关文件有改动，通常会忽略；如果同一个文件里有用户改动影响任务，我需要读清楚并在其基础上工作。

避免破坏性 Git 命令：不会自行使用 git reset --hard、git checkout -- 之类会丢改动的命令，除非你明确要求。

提交/暂存时应只包含相关文件：如果需要 commit/stage，应该精确选择本次任务文件，而不是 git add . 式地把无关改动带进去。
